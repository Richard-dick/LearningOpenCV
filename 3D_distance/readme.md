# 2D to 3D

此文件夹下实现一个功能, 将rgb和深度图像综合, 选取rgb上一点坐标`(u,v)`, 并获取对应点的深度d

从而生成相机坐标系中的 `(x,y,z)`

经分析, 大致分为以下三步:

1. 读取图像, 两张, 需要附带检测鼠标的像素点功能, 由于不知道实际过程中的双图像采集细节, 所以只能传入两张图像.
2. 读入内参矩阵, 转换, 给出摄像头坐标系下坐标
3. 生成xyz, 并尝试使用棋盘标定函数来测试是否准确.

> 一些小问题
> 1. 深度图的对应关系, 只记得6.7mm
> 2. 深度图和rgb图左侧有部分像素没对齐
> 3. 两个图貌似不是同时存储的, 这点会有影响吗

最后是写一个检测函数评估一下即可
